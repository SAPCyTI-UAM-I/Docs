# Ingreso al sistema SAPCyTI

## 1. Tarjeta

**ID:** HU-01

**Título:** Ingreso al sistema SAPCyTI

**Como:** usuario del sistema (administrador, profesor u otro rol autorizado)

**Quiero:** ingresar al sistema mediante mi login y contraseña

**Para:** acceder a la pantalla principal con las opciones correspondientes a mi tipo de usuario

## 2. Conversación

- **Acceso y Seguridad Inicial:** El usuario ingresa a la URL oficial https://paoscalu.izt.uam.mx/sapcyti. Debido a la infraestructura de hospedaje (PC de escritorio), el navegador presentará una advertencia de certificado de seguridad que el usuario debe aceptar explícitamente para visualizar la interfaz.

- **Interfaz de Autenticación:** La pantalla debe presentar los siguientes elementos obligatorios:
  - Campo de texto para Email.
  - Campo de tipo password para Contraseña.
  - Checkbox No cerrar sesión para persistencia de la sesión.
  - Botón ENVIAR para procesar y botón LIMPIAR para resetear campos.
  - Enlace "¿Olvidaste tu contraseña?" que redirige al flujo de recuperación mediante correo electrónico.

- **Validación y Roles**: El sistema debe validar que las credenciales existan en la base de datos y estén activas. Al ser exitoso, el sistema debe redirigir al Dashboard Principal mostrando únicamente las opciones de menú permitidas para el rol detectado: Coordinador, Profesor, Alumno, Asistente o Ponente.

- **Manejo de Errores**: En caso de credenciales inválidas, el sistema debe mostrar un mensaje de error genérico por seguridad y no permitir el avance a las funciones internas.

## 3. Criterios de Aceptación

```gherkin
Feature: Autenticación de usuarios en el sistema SAPCyTI

  Background:
    Given que el usuario navega a la URL "https://paoscalu.izt.uam.mx/sapcyti"
    And acepta la advertencia del certificado de seguridad del sitio

  @HappyPath
  Scenario: Acceso exitoso al sistema
    When ingresa un email registrado y una contraseña correcta
    And presiona el botón "ENVIAR"
    Then el sistema valida las credenciales exitosamente
    And redirige a la pantalla de inicio
    And despliega el menú lateral con las opciones específicas para su rol (ej. Coordinador, Alumno)

  @BadPath
  Scenario: Intento de acceso con credenciales incorrectas
    When ingresa un email o contraseña que no coinciden con los registros
    And presiona el botón "ENVIAR"
    Then el sistema muestra el mensaje de error "Credenciales inválidas"
    And permanece en la pantalla de login con los campos resaltados en error

  @BadPath
  Scenario: Intento de acceso con campos obligatorios vacíos
    When deja el campo "Email" o "Contraseña" vacío
    And presiona el botón "ENVIAR"
    Then el sistema impide la petición al servidor
    And muestra un mensaje indicando que los campos son obligatorios

  Scenario: Limpieza de formulario
    Given que el usuario ha escrito información en los campos de login
    When presiona el botón "LIMPIAR"
    Then el sistema debe borrar instantáneamente todo el texto capturado en los campos "Email" y "Contraseña"

  Scenario: Navegación a recuperación de acceso
    When selecciona la liga "¿Olvidaste tu contraseña?"
    Then el sistema debe mostrar la pantalla de "Recupera Contraseña" solicitando el email del usuario
```
