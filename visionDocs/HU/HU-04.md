# Edición de plan anual

## 1. Tarjeta
**ID:** HU-04

**Título:** Edición de plan anual

**Como:** coordinador del programa

**Quiero:** editar un plan anual seleccionando el año correspondiente

**Para:** gestionar y actualizar los planes trimestrales preliminares asociados

## 2. Conversación
* El coordinador accede al sistema con su usuario y contraseña.
* En el menú lateral selecciona la opción **“Editar plan anual”** dentro del módulo **Planeación Anual**.
* El sistema solicita seleccionar el año correspondiente.
* Una vez seleccionado el año:

  * El sistema muestra la pantalla con los trimestres preliminares asociados al plan anual.
  * Si los planes trimestrales ya fueron editados previamente, el sistema muestra un mensaje indicándolo.
* En la pantalla se muestran los trimestres con la opción **“Editar”**.
* Al presionar el botón **“Editar”** de un trimestre:

  * El sistema muestra la ventana de edición del plan trimestral preliminar.
  * Se despliega el listado de materias disponibles.
  * El coordinador puede:

    * Seleccionar las materias que se abrirán.
    * Definir el número de grupos.
    * Definir el cupo por grupo.
* El sistema debe permitir guardar los cambios realizados.


## 3. Criterios de Aceptación
```gherkin
Feature: Edición de Plan Anual

  Scenario: Acceso a la edición de plan anual
    Given que el coordinador ha iniciado sesión correctamente
    When selecciona la opción "Editar plan anual"
    And selecciona un año válido
    Then el sistema muestra los planes trimestrales preliminares asociados al año seleccionado

    Given que el coordinador selecciona un año cuyo plan anual ya fue editado previamente
    When el sistema muestra los trimestres asociados
    Then el sistema despliega un mensaje indicando que los planes trimestrales ya han sido editados

  Scenario: Ingreso a la edición de un trimestre
    Given que el coordinador visualiza los trimestres preliminares del plan anual
    When presiona el botón "Editar" en un trimestre
    Then el sistema muestra la ventana de edición del plan trimestral preliminar
    And presenta la lista de materias disponibles

  Scenario: Configuración de materias en el trimestre
    Given que el coordinador se encuentra en la ventana de edición del trimestre
    When selecciona las materias que se abrirán
    And define el número de grupos
    And define el cupo por grupo
    And guarda los cambios
    Then el sistema registra la información correctamente
    And confirma que la actualización fue exitosa

  Scenario: Validación de datos en la edición trimestral
    Given que el coordinador se encuentra editando un trimestre
    When ingresa valores inválidos en número de grupos o cupo
    Then el sistema muestra un mensaje de error
    And no permite guardar hasta corregir los datos
```
