# Imprimir formato de inscripción

## 1. Tarjeta

**ID:** HU-09
**Título:** Generación y descarga del formato PDF de inscripción (Solicitud de UEA).

**Como:** Coordinador o Asistente del Posgrado.
**Quiero:** Seleccionar a un alumno cuya carga académica ya fue aprobada para generar su formato de inscripción.
**Para:** Obtener el documento oficial en PDF que formaliza el registro de materias ante Sistemas Escolares.

## 2. Conversación

- **Coordinador:** Una vez que el profesor aprueba las materias, el sistema me envía una notificación por correo.
- **Sistema:** Al entrar a la sección "Imprimir formato de inscripcion", verás una lista de alumnos listos para procesar.
- **Coordinador:** Busco al alumno (por ejemplo, Paulina Valencia Franco) y presiono el botón "Imprimir".
- **Sistema:** Se genera una vista previa o descarga de un archivo PDF llamado "SOLICITUD DE UEA - GRUPO A ESTUDIOS DE POSGRADO".
- **Asistente:** El PDF debe contener todos los datos del alumno (Matrícula, División, Posgrado) y el desglose de las UEA con su clave, créditos y grupo.
- **Sistema:** Además, el sistema lleva un control de quiénes ya tienen su horario impreso en una sección aparte.

## 3. Criterios de Aceptación

```gherkin
Feature: Imprimir formato de inscripción

  Scenario: Visualización de alumnos pendientes de impresión
    Given que el coordinador ha recibido la notificación de aprobación por correo
    When accede a la opción "Imprimir formato de inscripcion" del menú "PROCESO DE INSCRIPCION"
    Then el sistema muestra la tabla "Seleccione el alumno para imprimir su inscripcion"
    And despliega el nombre del alumno con un botón "Imprimir" asociado

  Scenario: Generación exitosa del formato PDF
    Given que el alumno aparece en la lista de pendientes de impresión
    When el coordinador presiona el botón "Imprimir"
    Then el sistema abre una nueva pestaña o descarga un archivo PDF
    And el documento se titula "SOLICITUD DE UEA - GRUPO A ESTUDIOS DE POSGRADO"

  Scenario: Validación de datos en el documento PDF
    Given que se ha generado el PDF para un alumno específico
    Then el documento debe mostrar correctamente:
      | Campo         | Ejemplo de Valor                  |
      | Nombre        | Valencia Franco Paulina          |
      | Matrícula     | 2123803361                       |
      | Posgrado      | CIENCIAS Y TECNOLOGÍAS DE LA INF.|
      | Trimestre     | 16I                              |
    And debe incluir la tabla de UEA con Clave, Créditos (ej. 9) y Grupo (ej. CP33)

  Scenario: Control de alumnos con formato ya impreso
    Given que un formato de inscripción ya ha sido generado previamente
    When el coordinador consulta la pantalla de impresión
    Then el nombre del alumno debe aparecer en la sección "Alumnos con horario Impreso"
    And permitir la re-impresión del formato si es necesario mediante el botón "Imprimir"

  Scenario: Marcas de validación en el PDF
    Given el formato PDF generado
    Then el sistema debe marcar con una "X" los recuadros correspondientes a:
      | Sección      | Opción marcada  |
      | División     | CBI             |
      | Nivel        | MAESTRIA        |
    And presentar los espacios para firma del "SOLICITANTE" y "COORDINACION DE SISTEMAS ESCOLARES"
```
