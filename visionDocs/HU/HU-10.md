# Editar estado del proceso de inscripción

## 1. Tarjeta 

**ID:** HU-10

**Título:** Gestión administrativa y modificación de estados de inscripción.

**Como:** Coordinador del Posgrado.

**Quiero:** Modificar manualmente el estado del proceso de inscripción de cualquier alumno y visualizar un resumen general.

**Para:** Corregir errores en el flujo (como permitir que un alumno vuelva a elegir materias) o dar por finalizado el proceso de manera administrativa.


## 2. Conversación 

* **Coordinador:** A veces un alumno se equivoca o el profesor le pide cambiar algo cuando ya imprimimos el formato. Necesito poder "regresarlos" en el sistema.
* **Sistema:** Para eso está la pantalla "Editar estado del proceso de inscripción". Muestra a los alumnos con un selector de estado.
* **Coordinador:** Veo que los estados en el menú desplegable coinciden con las etapas: Selección, Aprobación, Impresión y Terminado.
* **Sistema:** Exacto. Solo eliges el nuevo estado y presionas el botón "Asignar Estado" para que el sistema actualice el flujo de ese alumno.
* **Coordinador:** ¿Y cómo sé cuántos alumnos hay en cada etapa sin buscarlos uno por uno?
* **Sistema:** En la parte inferior de la misma pantalla, el sistema agrupa a los alumnos automáticamente bajo encabezados como "-- Aprobación de Materias --" o "-- Terminado --" para darte ese resumen visual.



## 3. Criterios de Aceptación

```gherkin
Feature: Editar estado del proceso de inscripción

  Scenario: Visualización del listado maestro de alumnos
    Given que el coordinador accede a la opción "Editar estado del proceso de inscripción"
    Then el sistema muestra una tabla con el número y nombre completo de todos los alumnos registrados en el ciclo actual
    And presenta un menú desplegable (combo-box) y un botón "Asignar Estado" para cada registro

  Scenario: Modificación manual del estado de un alumno
    Given que el coordinador identifica a un alumno que requiere un cambio de fase
    When selecciona una de las siguientes opciones en el menú desplegable:
      | Estado      |
      | Seleccion   |
      | Aprobacion  |
      | Impresion   |
      | Terminado   |
    And presiona el botón "Asignar Estado"
    Then el sistema actualiza inmediatamente los permisos del alumno para reflejar la fase elegida

  Scenario: Visualización del resumen por etapas (Dashboard inferior)
    Given que el coordinador se encuentra en la pantalla de edición de estados
    When se desplaza a la parte inferior de la página
    Then el sistema muestra las secciones de resumen:
      | Sección                     |
      | -- Aprobación de Materias --|
      | -- Impresion --             |
      | -- Terminado --             |
    And lista a los alumnos que pertenecen a cada categoría para un conteo rápido

  Scenario: Reversión de estado para correcciones
    Given que un alumno ya estaba en la fase de "Terminado"
    When el coordinador cambia su estado a "Seleccion"
    Then el sistema debe permitir al alumno volver a ingresar al módulo de "Inscripción" para elegir materias nuevamente

  Scenario: Persistencia del cambio de estado
    Given que se ha asignado un nuevo estado a un alumno
    When el coordinador navega a otras secciones del sistema y regresa
    Then el alumno debe permanecer en la última categoría asignada manualmente
```
