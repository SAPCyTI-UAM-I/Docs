# Implementación — SAPCyTI

## Propósito

Estado operativo del proyecto: plan de fases, memoria de decisiones y mapas de tareas (`phaseX.md`).

## Cuándo leer esto

- Inicio de cada sesión de trabajo
- Para saber la fase actual y la siguiente spec
- Para registrar decisiones (Decision Log)

## Índice

| Archivo | Contenido | Obligatorio |
|---------|-----------|-------------|
| [implementationPlan.md](implementationPlan.md) | Fases 0–9, dependencias, vista macro | Referencia |
| [progress.md](progress.md) | Estado fino, Current Phase, Decision Log | **Sí** (inicio de sesión) |
| [phase0.md](phase0.md) … [phase9.md](phase9.md) | Tareas y links a specs | Según fase asignada |
| [example/](example/) | Plantillas de ejemplo | No |

## Rutas canónicas

- Estado fino: `progress.md` (prevalece sobre `implementationPlan.md` en conflictos)
- Specs: [`../SDD/specs/`](../SDD/specs/)

## Qué NO cargar al implementar

- `implementationPlan.md` completo si ya tienes la spec
- Fases distintas a la asignada

## Siguiente paso

Leer `progress.md` § Current Phase → abrir `phaseX.md` → spec enlazada.
