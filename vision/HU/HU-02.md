# Recuperación de contraseña

## 1. Tarjeta

**ID:** HU-02

**Título:** Recuperación de contraseña

**Como:** usuario registrado del sistema SAPCyTI

**Quiero:** recuperar mi contraseña cuando la haya olvidado

**Para:** poder restablecer mi acceso al sistema

## 2. Conversación

- **Punto de Entrada:** El acceso a este flujo es exclusivamente a través del enlace **“¿Olvidaste tu contraseña?”** ubicado en la pantalla de autenticación principal (HU-01).
- **Interfaz de Usuario:** El sistema redirige a una vista simplificada titulada **"Recupera Contraseña"**. Esta pantalla debe contener:
  - Una etiqueta de instrucción: "Escriba su dirección de correo electrónico para recuperar su contraseña".
  - Un campo de entrada de texto etiquetado como **Email**.
  - Un botón de acción principal etiquetado como **Enviar Correo**.
  - Un enlace o botón de navegación etiquetado como **Inicio** para cancelar la operación y volver al login.

- **Lógica de Negocio y Validación:**
  - El sistema debe verificar que el campo no esté vacío y que el texto ingresado cumpla con el estándar de una dirección de correo electrónico (presencia de `@` y dominio).
  - **Validación de Existencia:** El sistema consulta la base de datos de usuarios (Alumnos, Profesores, Coordinadores).
  - **Seguridad:**
    - Si el correo existe, el sistema genera un token de seguridad y dispara un proceso de envío de correo para la recuperación de contraseña. Se muestra el mensaje "Un correo electrónico con instrucciones para recuperar su contraseña ha sido enviado a la dirección indicada, si es que existe en el sistema".
    - Si el correo no existe, el sistema no genera nada y no informa al usuario de que su correo no existe. Se muestra el mensaje "Un correo electrónico con instrucciones para recuperar su contraseña ha sido enviado a la dirección indicada, si es que existe en el sistema".

## 3. Criterios de Aceptación

```gherkin
Feature: Recuperación de contraseña en SAPCyTI

  Background:
    Given que el usuario no recuerda su clave de acceso
    And se encuentra en la pantalla de "Recupera Contraseña"

  @HappyPath @Security
  Scenario: Solicitud de recuperación con correo registrado
    When ingresa el correo electrónico "usuario.valido@uam.mx" que está registrado en el sistema
    And presiona el botón "Enviar Correo"
    Then el sistema debe generar un token de seguridad internamente
    And realizar el envío del mensaje de recuperación
    And mostrar el mensaje: "Un correo electrónico con instrucciones para recuperar su contraseña ha sido enviado a la dirección indicada, si es que existe en el sistema"

  @BadPath @Security
  Scenario: Solicitud de recuperación con correo no registrado
    When ingresa el correo electrónico "desconocido@uam.mx" que no existe en la base de datos
    And presiona el botón "Enviar Correo"
    Then el sistema no debe generar ningún token ni realizar envíos de correo
    And debe mostrar exactamente el mismo mensaje: "Un correo electrónico con instrucciones para recuperar su contraseña ha sido enviado a la dirección indicada, si es que existe en el sistema"

  @BadPath
  Scenario: Validación de formato de correo inválido
    When ingresa un texto con formato incorrecto (ej: "usuario_sin_arroba")
    And presiona el botón "Enviar Correo"
    Then el sistema debe impedir la petición
    And mostrar una alerta indicando que el formato del correo es incorrecto

  @BadPath
  Scenario: Validación de campo obligatorio vacío
    When deja el campo "Email" vacío
    And presiona el botón "Enviar Correo"
    Then el sistema debe resaltar el campo como obligatorio
    And no debe procesar ninguna validación de servidor

  @Navigation
  Scenario: Cancelar y regresar al inicio
    When selecciona la opción "Inicio"
    Then el sistema debe redirigir al usuario a la pantalla de login (HU-01)
    And asegurar que no se realizó ninguna acción de recuperación
```
