# Cerrar sesión

## 1. Tarjeta

**ID:** HU-03

**Título:** Cerrar sesión

**Como:** usuario autenticado del sistema SAPCyTI

**Quiero:** cerrar mi sesión cuando termine de utilizar el sistema

**Para:** proteger mi información y evitar accesos no autorizados

## 2. Conversación

- **Ubicación del control:**
  - Una vez que el usuario ha iniciado sesión correctamente, el sistema muestra la pantalla principal.

  - El botón para finalizar la sesión se etiqueta como **"Cerrar sesión"** y se encuentra ubicado en la parte inferior del menú lateral izquierdo.

- **Acción del sistema:** Al presionar el botón, el sistema debe:
  - Notificar al servidor para invalidar el token de sesión.
  - Limpiar cualquier dato del usuario almacenado en la memoria volátil del navegador (sessionStorage/localStorage).
  - Ignorar la persistencia si se marcó la opción "No cerrar sesión" en el login previo, forzando la salida total.

- **Redirección:** El sistema debe redirigir al usuario inmediatamente a la pantalla de Login (HU-01).

- **Seguridad del Navegador:** Se debe configurar el sistema para que las páginas protegidas no se almacenen en la caché del navegador, asegurando que el botón "Atrás" no renderice información sensible tras el cierre.

- **Control de Acceso:** Cualquier intento de navegación manual a una ruta protegida (ej. `/sapcyti/inscripcion`) sin una sesión activa debe ser interceptado y redirigido al login.

## 3. Criterios de Aceptación

```gherkin
Feature: Cierre de sesión en SAPCyTI

  Background:
    Given que el usuario ha iniciado sesión y tiene acceso al menú lateral

  @HappyPath @Navigation
  Scenario: Cierre de sesión exitoso desde el menú
    When selecciona el botón "Salir" en la parte inferior del menú izquierdo
    Then el sistema debe invalidar la sesión en el servidor
    And redirigir al usuario a la pantalla de login
    And mostrar un mensaje opcional de "Sesión finalizada correctamente"

  @Security
  Scenario: Intento de acceso a ruta protegida sin sesión activa
    Given que el usuario no tiene una sesión activa o acaba de cerrarla
    When intenta acceder directamente a la URL "[https://paoscalu.izt.uam.mx/sapcyti/listaAlumnos](https://paoscalu.izt.uam.mx/sapcyti/listaAlumnos)"
    Then el sistema debe interceptar la petición
    And redirigir automáticamente a la pantalla de login
    And no mostrar ningún dato del contenido protegido

  @Security @BrowserBehavior
  Scenario: Uso del botón "Atrás" del navegador tras cerrar sesión
    Given que el usuario ha cerrado sesión exitosamente
    And visualiza la pantalla de login
    When presiona el botón "Atrás" del navegador
    Then el sistema no debe mostrar la pantalla anterior con datos
    And debe mantener al usuario en la pantalla de login o mostrar un error de "Sesión expirada"

  @Security
  Scenario: Invalidación de la persistencia "No cerrar sesión"
    Given que el usuario ingresó previamente marcando "No cerrar sesión"
    When presiona el botón "Salir"
    Then el sistema debe eliminar las cookies de persistencia (remember-me)
    And solicitar credenciales completas en el siguiente intento de ingreso
```
